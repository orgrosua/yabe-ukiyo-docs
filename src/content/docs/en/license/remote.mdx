The remote feature complements the [license system](../license). It allows you to manage Bricks 3rd-party remote templates with ease.

If you have a password or license key from a 3rd-party template library, you can add it to the Remote list, which will be available in the Bricks editor.

## Adding a 3rd-party remote template

### Step 1: Add new remote

Go to **Bricks â†’ Yabe Ukiyo**. Switch to the **Remotes** tab and click the **Add New** button.

<a data-fslightbox href="/tutorial/license/remote/1.png">
![Add new remote to yabe ukiyo](/tutorial/license/remote/1.png)
</a>

### Step 2: Provide remote details

Fill in the form with the remote details. The **License Key** field can be filled with the password or license key you received from the 3rd-party template library.

If you have [copied](../license/#:~:text=copy%20the%20token%20to%20your%20clipboard) the **Token** from the Yabe Ukiyo license system, you can click the **Paste Token** button to automatically fill all the fields.

<a data-fslightbox href="/tutorial/license/remote/2.png">
![Enter the license Key](/tutorial/license/remote/2.png)
</a>

Then click the **Save** button.

<a data-fslightbox href="/tutorial/license/remote/3.png">
![Remote list](/tutorial/license/remote/3.png)
</a>

### Step 3: Using the 3rd-party template library

Now you can use the 3rd-party template library in the Bricks editor.
Make sure to hard-refresh the template libaries by clicking the **Refresh** button to load the new remote template.

<a data-fslightbox href="/tutorial/license/remote/4.png">
![Using the 3rd-party template library](/tutorial/license/remote/4.png)
</a>
